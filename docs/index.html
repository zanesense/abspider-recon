Usage Guide: Starting a Scan

Scan Execution Workflow

  1. 1

    Target and Modules Selection

    Click "New Scan" in the sidebar, enter the target domain or URL (e.g., example.com), and select the reconnaissance modules you want to run.

  2. 2

    Concurrency and Proxy Setup

    Set Threads (1-50) for concurrency. Enable Use Proxy Rotation if you have configured proxy servers in the Settings panel.

  3. 3

    Start and Monitor

    Click "Start Reconnaissance". Scans appear on the Dashboard immediately and update in real-time.

  4. 4

    Viewing Results

    Scans appear on the Dashboard immediately. Click any scan to view detailed results. You'll see real-time updates while the scan is running and a progress indicator showing the current stage. Results are organized by module.

Configuration & Reporting

  • A

    Report Generation

    Open the completed scan, click "Download Report", and the PDF automatically downloads with all findings. Includes headers, WHOIS, subdomains, ports, and vulnerability assessments.

  • B

    Discord Notifications

    Go to Settings, enter your Discord webhook URL, and click "Test Webhook" to verify. Once configured, use the "Send to Discord" button on scan results to get real-time notifications when scans complete.

  • C

    Proxy Setup

    In Settings, enter your proxy list (one per line, e.g., http://proxy1.example.com:8080) and click "Save Settings". Enable "Use Proxy Rotation" when starting scans to help avoid rate limiting.

Troubleshooting & Known Issues

Scan Fails Immediately

  • Check target URL format.
  • Ensure the target is accessible.
  • Try with fewer modules enabled.
  • Check browser console for errors.

No Subdomains Found

  • Target may have few subdomains.
  • DNS queries may be rate-limited.
  • Try again with proxy rotation.
  • Some domains block enumeration.

Headers Not Loading

  • Target may block HEAD requests.
  • CORS may prevent access.
  • Try with proxy enabled.
  • Some sites require authentication.

Discord Webhook Not Working

  • Verify webhook URL is correct.
  • Use "Test Webhook" button first.
  • Check Discord server permissions.
  • Ensure webhook hasn't been deleted.

DDoS/WAF Detection Issues

  • Some WAFs are designed to be stealthy and may not be easily detected.
  • Aggressive scanning might trigger rate limits or temporary blocks.
  • Ensure your proxy configuration is correct if using one.

Technical Specifications

Data Architecture

  • Storage: All scans stored in browser localStorage.
  • Serverless: Persists across sessions; no server-side storage.
  • Performance: Start with 5 threads, but increase up to 50 for faster scans.
  • Clear browser data to reset all stored scans and settings.

Integrated APIs

ABSpider leverages multiple public APIs for data enrichment:

  • Google DNS over HTTPS: Used for subdomain enumeration and DNS lookups.
  • crt.sh: Certificate Transparency logs for subdomain discovery.
  • RDAP: Domain registration data (WHOIS fallback).
  • ipapi.co / ip-api.com: IP and domain geolocation information.
  • Optional APIs: Shodan, VirusTotal, SecurityTrails, BuiltWith, OpenCage, Hunter.io, Clearbit (configured in settings for enhanced data).

Browser Limitations & Performance

  • CORS restrictions may limit some scans, which is mitigated by the built-in CORS bypass.
  • Port scanning uses browser-based connectivity checks (not raw sockets), which may not be as accurate as server-side tools.
  • Some targets may block automated requests or implement advanced bot detection.
  • Rate limiting may occur on aggressive scans; use proxy rotation and adjust thread count.
  • Performance Tips: Start with 5 threads (default). Increase threads for faster scans (up to 50). Use proxy rotation for large scans. Disable unused modules to speed up scans.

Best Practices

Responsible Usage

  • Start small - Test with well-known domains first.
  • Respect rate limits - Don't scan too aggressively.
  • Use responsibly - Only scan domains you own or have explicit written permission for.
  • Save important scans - Download PDF reports for critical findings.
  • Configure Discord - Get notified when long scans complete.

Security Considerations

  • Client-Side API Keys: Be aware that optional API keys are stored in your browser's local storage. This is insecure for sensitive keys. Use only for testing or public keys.
  • Public CORS Proxies: Using public proxies can expose your request details. Consider self-hosting a trusted proxy for sensitive operations.
  • Regular Updates: Keep your browser and the application up-to-date to benefit from the latest security patches and features.
