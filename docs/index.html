Usage Guide: Starting a Scan

Scan Execution Workflow

  1. 1

    Target and Modules Selection

    Click "New Scan" in the sidebar, enter the target domain or URL (e.g., example.com), and select the reconnaissance modules you want to run.

  2. 2

    Concurrency and Proxy Setup

    Set Threads (1-100) for concurrency. Enable Use Proxy Rotation if you have configured proxy servers in the Settings panel.

  3. 3

    Start and Monitor

    Click "Start Reconnaissance". Scans appear on the Dashboard immediately and update in real-time.

Configuration & Reporting

  • A

    Report Generation

    Open the completed scan, click "Download Report", and the PDF automatically downloads with all findings.

  • B

    Discord Notifications

    Go to Settings, enter the Discord webhook URL, and use the "Test Webhook" button to verify.

  • C

    Proxy Setup

    In Settings, enter your proxy list (one per line) and click "Save Settings".

Troubleshooting & Known Issues

Scan Fails Immediately

  • Check target URL format.
  • Ensure the target is accessible.
  • Try with fewer modules enabled.
  • Check browser console for errors.

No Subdomains Found

  • Target may have few subdomains.
  • DNS queries may be rate-limited.
  • Try again with proxy rotation.
  • Some domains block enumeration.

Headers Not Loading

  • Target may block HEAD requests.
  • CORS may prevent access.
  • Try with proxy enabled.
  • Some sites require authentication.

Discord Webhook Not Working

  • Verify webhook URL is correct.
  • Use "Test Webhook" button first.
  • Check Discord server permissions.
  • Ensure webhook hasn't been deleted.

Technical Specifications

Data Architecture

  • Storage: All scans stored in browser localStorage.
  • Serverless: Persists across sessions; no server-side storage.
  • Performance: Start with 5 threads, but increase up to 20 for faster scans.

Integrated APIs

ABSpider leverages multiple public APIs for data enrichment:

  • Google DNS over HTTPS: Used for subdomain enumeration.
  • crt.sh: Certificate Transparency logs.
  • RDAP: Domain registration data.
  • ipapi.co: IP and domain information.
