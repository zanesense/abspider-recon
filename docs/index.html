What is ABSpider Recon?

A cutting-edge, browser-based reconnaissance dashboard crafted for security professionals, penetration testers, and bug bounty hunters. Built with Vite + React for unparalleled performance.

Security First

Supabase-powered magic link authentication eliminates password management. Row-level security ensures your data stays protected.

Intelligence-First Approach

Prioritizes passive data collection (WHOIS, CT Logs, GeoIP) to minimize footprint, with powerful active scans available when authorized.

Modular Architecture

20+ reconnaissance modules covering DNS, WHOIS, subdomains, ports, vulnerabilities, and more. Use what you need, when you need it.

CORS & Cloudflare Bypass

Intelligent bypass mechanisms overcome common web protections, enabling comprehensive reconnaissance even on protected targets.

Smart Scan Manager

AI-powered target analysis with adaptive payload management. Automatically adjusts scan intensity based on target health and response.

User Management

Rich profile cards, comprehensive settings, notification preferences, and data export capabilities for complete account control.

Quick Installation

Get up and running in minutes with our simple installation process.

Installation Steps

1. Clone the repository

git clone https://github.com/zanesense/abspider-recon.git cd abspider-recon

2. Install dependencies

npm install

3. Configure environment

cp .env.example .env # Edit .env with your Supabase credentials

4. Start development server

npm run dev

The application will be available at http://localhost:5000

Reconnaissance Modules

Comprehensive suite of passive and active reconnaissance modules for thorough security analysis.

Passive Reconnaissance Low Risk

Site Information

Basic website details, IP address, web server, CMS, and robots.txt analysis

HTTP Headers Analysis

Real-time security header scoring (HSTS, CSP, XFO) and technology fingerprinting

Tech Stack Fingerprinting

Identifies web technologies, frameworks, and analytics used by the target

WHOIS / RDAP Lookup

Domain registration, registrar, nameservers, and date information

GeoIP Location

Pinpoints the physical location of the target server's IP address

DNS Records

Enumerates A, AAAA, CNAME, TXT, MX, NS, and SOA records

MX Records

Analyzes mail server configurations, including SPF and DMARC

Subdomain Enumeration

Discovers subdomains using Certificate Transparency logs and DNS lookups

Reverse IP Lookup

Identifies other domains hosted on the same IP address

SEO Analysis

Analyzes meta tags, headings, links, and page performance for SEO insights

SSL/TLS Analysis

Analyzes SSL/TLS certificate details, issuer, expiry, and common names

Active Reconnaissance Authorization Required

Port Scanning

Checks connectivity and identifies services on common ports, with Shodan integration

SQL Injection Test

Checks for potential SQL Injection (SQLi) vulnerabilities with real payloads

XSS Detection

Detects reflected, DOM, and stored Cross-Site Scripting (XSS) vulnerabilities

LFI Scanning

Scans for Local File Inclusion vulnerabilities using real payloads

CORS Misconfiguration

Identifies Cross-Origin Resource Sharing (CORS) vulnerabilities

VirusTotal Scan

Domain reputation, malware scanning, and threat intelligence via VirusTotal API

Broken Link Checker

Scans for broken internal and external links on the target website

DDoS Firewall Test

Detects WAF/DDoS protection mechanisms (e.g., Cloudflare, Sucuri)

Utility Modules Helper Tools

Subnet Scan

Calculates network range details for a given IP and CIDR

WordPress Scan

Identifies WordPress versions, plugins, themes, and common vulnerabilities

Legal Notice

Active scanning modules require explicit written authorization. Only scan systems you own or have permission to test. Unauthorized scanning may be illegal in your jurisdiction.

Optional API Integrations

Enhance your reconnaissance capabilities with third-party API integrations for deeper insights.

⚠️ Client-Side API Key Storage Warning

API keys are stored in your Supabase database but are still accessible client-side. Any XSS vulnerability or physical access to your browser could expose these keys.

DO NOT store sensitive, paid, or production API keys here. This feature is intended for testing with non-critical keys only.
