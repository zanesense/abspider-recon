Troubleshooting

The most common ABSpider Recon issues, with the exact command or check that resolves them. If you do not see your issue here, open a ticket on GitHub Issues.

Installation

Node.js version error

Symptom: Error: The engine "node" is incompatible with this module

Fix:

node --version # should print v20.x or higher

If you have an older Node, install Node 20 LTS from nodejs.org or via nvm:

nvm install 20 nvm use 20

npm install fails

Symptom: peer-dependency errors, lockfile mismatch, or network timeouts.

# Clean install rm -rf node_modules package-lock.json npm cache clean --force npm install # Or use the committed lockfile exactly npm ci

Port 5000 already in use

Symptom: Error: listen EADDRINUSE: address already in use :::5000

# Windows netstat -ano | findstr :5000 taskkill /PID <PID> /F # macOS / Linux lsof -ti:5000 | xargs kill -9

Or change the port in vite.config.ts.

Stale TypeScript build cache

Symptom: tsc -b reports errors that don't match the source.

# Windows PowerShell Remove-Item tsconfig.app.tsbuildinfo, tsconfig.node.tsbuildinfo -ErrorAction SilentlyContinue Remove-Item -Recurse -Force dist -ErrorAction SilentlyContinue npm run typecheck

Authentication

Magic link not received

  1. Check your spam / junk folder.
  2. Verify the email address is correct.
  3. Wait 2–3 minutes — Supabase emails can be delayed during peak hours.
  4. Confirm the project is active in the Supabase dashboard.
  5. Try a different email address.

Magic link expired

Links expire after 1 hour. Request a new one and click it immediately. If you clicked an old link, request a fresh one — old links are invalidated as soon as a new one is sent.

Session keeps expiring

JWTs default to a 7-day expiry. To change it, edit Supabase → Authentication → Settings → JWT expiry. The dashboard listens for SIGNED_OUT events and force-redirects to /login if a session is lost in another tab.

Sign-in works but no rows are visible

The RLS policies from 0002_rls_policies.sql were not applied. Re-apply the migration in the Supabase SQL Editor, or run:

SELECT * FROM pg_policies WHERE tablename IN ('scans', 'user_settings', 'user_profiles', 'api_keys');

Scanning

Scan stuck at 0%

  1. Open the browser console (F12) and look for network errors.
  2. Verify the target is reachable from your network (try curl -I <target>).
  3. Check that the Supabase session is still valid.
  4. Disable browser extensions — ad blockers and privacy extensions are common culprits.
  5. Reduce the thread count or switch to conservative mode.

Target is not reachable

Confirm the target accepts incoming requests from your network. If you are behind a corporate proxy or VPN, the dashboard's fetch may be blocked. The CLI is unaffected because it runs in Node.js, not the browser.

CORS errors in the browser

Some targets reject cross-origin requests. The dashboard falls back to the Vercel Edge proxy automatically. If the proxy is also blocked (or you are self-hosting without it), use the CLI for that target.

Scan times out before completing

  • Increase the timeout in Settings → Scanning defaults.
  • Reduce the number of enabled modules.
  • Lower the thread count.
  • Try during off-peak hours if the target is rate-limited.

Rate limiting (HTTP 429)

  • Reduce threads; raise the request delay.
  • Enable proxy rotation in Settings → Proxy & Network.
  • Wait 5–10 minutes and retry.
  • Use --mode conservative on the CLI.

Incomplete results (some modules return no data)

  • The target may not support the feature (e.g. no robots.txt for the SEO module).
  • Verify the relevant API key is configured (Shodan, VirusTotal, etc.).
  • Some modules need an IP — if the target is behind a CDN, GeoIP and Subnet may not return useful data.

Performance

Scans are slow

  • Increase thread count (up to 50 in adaptive mode, 100 in aggressive).
  • Reduce per-request timeout.
  • Disable modules you don't need for this engagement.
  • Check your internet speed — the dashboard's bottleneck is often upstream bandwidth.

Browser is freezing

  • Reduce thread count.
  • Close other tabs to free memory.
  • Disable browser extensions (especially ad blockers and translation tools).
  • Run the scan in Chrome or Edge for best performance.

High memory usage

  • Reduce the scan history limit in Settings → Data Management.
  • Clear old scans from All Scans.
  • Lower the thread count.
  • Restart the browser periodically.

API integrations

Shodan module returns no enhanced data

  1. Verify the API key is correct in Settings → API Integration → Shodan.
  2. Check you have API credits remaining (Shodan shows usage at account.shodan.io).
  3. Test the key directly with curl:
    curl https://api.shodan.io/api-info?key=YOUR_API_KEY

VirusTotal times out or 429s

The free tier is rate-limited to 4 requests / minute and 500 / day. If you hit the limit:

  • Wait a minute between requests.
  • Disable the VirusTotal module for the rest of the day.
  • Upgrade to a paid plan if you need higher throughput.

SecurityTrails returns no historical data

The target may not have a public history on SecurityTrails. Verify the key works:

curl -H "APIKEY: YOUR_API_KEY" \ https://api.securitytrails.com/v1/ping

"Invalid API key" error

  1. Copy the key carefully — no extra spaces, no missing characters.
  2. Regenerate the key in the service's dashboard and try again.
  3. Check the key hasn't been revoked or expired.
  4. Confirm the right service is selected in the dashboard.

CORS & the Edge proxy

Proxy returns 500

The upstream fetch failed. Verify the URL is reachable from the Vercel Edge network (try curl on a different network). The proxy is not a generic web proxy — it does not bypass provider challenges.

Proxy returns 400

The url query parameter is missing or did not parse with new URL(). Make sure the URL is fully qualified (e.g. https://example.com/) and URL-encoded when passed in the query string.

Proxy is not bundled in my Docker image

By design — the production Docker image is a static Nginx server. To get the proxy on Docker, run the Vercel deployment in parallel, or port the 92-line api/proxy.ts handler to a sidecar container.

Error messages

"Failed to fetch"

Network connectivity issue. Check the target URL, your firewall, and your network. Try disabling your VPN temporarily.

"Supabase connection failed"

  1. Verify .env exists and contains the URL + anon key.
  2. Confirm the Supabase project is not paused.
  3. Check the Supabase status page.
  4. Try a different network.

"Module execution failed"

A specific module hit an unrecoverable error. The other modules still complete. Check the browser console for the underlying error and verify the relevant API key if the module needs one.

"Storage quota exceeded"

The browser's localStorage is full. Reduce the scan history limit in Settings → Data Management and clear old scans.

"Invalid target format"

The dashboard accepts base domains (example.com), full URLs (https://example.com), and IPs (192.168.1.1). It does not accept paths — use the base domain only.

"Unauthorized access"

  1. Sign out and sign in again.
  2. Clear browser cookies and cache.
  3. Check the session hasn't expired.
  4. Verify the RLS policies are in place.

Database

RLS policy error

Symptom: new row violates row-level security policy

Fix: verify the policies exist and re-apply migration 0002_rls_policies.sql if any are missing.

Data not persisting

  1. Confirm the Supabase session is active.
  2. Open the browser dev tools and check for network errors to *.supabase.co.
  3. Verify the RLS policies allow the operation for the current user.
  4. Try a different browser profile.

Proxy

Proxy connection failed

  1. Verify the proxy URL format (scheme://host:port).
  2. Test the proxy with curl from a terminal.
  3. Check proxy authentication credentials.
  4. Ensure the proxy supports HTTPS if you are scanning HTTPS targets.

Proxy rotation not working

  1. Verify multiple proxies are configured.
  2. Check the rotation strategy (round-robin or random).
  3. Test each proxy individually with the Test Proxies button.
  4. Confirm "fallback to direct" is enabled if all proxies fail.

Discord webhooks

No notifications received

  1. Verify the webhook URL is complete (starts with https://discord.com/api/webhooks/).
  2. Test the webhook directly in Discord to confirm it works.
  3. Check the channel permissions.
  4. Confirm the webhook hasn't been deleted from the Discord server.

"Invalid webhook format"

The URL must be in the form https://discord.com/api/webhooks/ID/TOKEN. Common mistakes: missing the protocol, incomplete token, extra whitespace, or the wrong domain.

Collecting debug information

Before opening a GitHub issue, collect the following:

  1. Browser console — press F12, go to Console, copy the errors.
  2. Network tab — reproduce the issue, then look for failed requests.
  3. System information:
    node --version npm --version abspider --version # Browser: Chrome 120, Firefox 121, Safari 17, …
  4. Environment — local / Vercel / Docker / self-hosted.

Emergency fixes

Complete local reset

// ⚠️ Use with caution — this clears all local data localStorage.clear(); sessionStorage.clear(); // Refresh the page

Factory reset from the UI

  1. Open Settings → Data Management.
  2. Click Reset to Defaults and confirm.
  3. Sign out and sign in again.
