CLI Reference

The abspider npm package ships the same scanner engine as the dashboard, packaged for the terminal. It runs without a Supabase session, supports graceful Ctrl+C, and emits structured JSON for downstream tooling.

Synopsis

abspider <target> [options] # Examples npx abspider example.com --all --mode aggressive abspider https://staging.example.com --modules dns,sslTls,ports --output scan.json abspider example.com --json --pretty | jq .

The target accepts a domain (example.com), a hostname (api.example.com), or a full URL (https://example.com:8443/path). The URL form is used to derive scheme, port, and base path.

Installation

# One-off npx abspider example.com # Global install npm install -g abspider abspider example.com
bunx abspider example.com # Global install bun add -g abspider abspider example.com
git clone https://github.com/zanesense/abspider-recon.git cd abspider-recon npm install npm run cli -- example.com

Local dev uses the source tree under packages/cli. The published abspider tarball bundles a frozen copy of the same modules.

Module selection

Use --modules (or -m) to run an explicit set. Module names are case-insensitive and a few common aliases are accepted (site → siteInfo, ssl → sslTls, header → headers, tech → techStack).

# Passive + a few active abspider example.com --modules siteInfo,headers,whois,dns,sslTls,ports,corsMisconfig # Just infrastructure abspider example.com -m dns,mx,subdomains,reverseip # Everything (equivalent to --all) abspider example.com -m siteInfo,headers,whois,geoip,dns,mx,subnet,subdomains,reverseip,virustotal,sslTls,techStack,seo,ports,sqlinjection,xss,lfi,wordpress,brokenLinks,corsMisconfig,ddosFirewall

Scan modes

Each mode tunes the per-module defaults. You can override any value with the corresponding flag — the override is sticky for the rest of the run.

Adaptive mode behavior

In adaptive mode (the default) the CLI watches the timing of each module. If a module takes longer than 75% of the timeout (with a 2.5s floor), the runner treats the target as overloaded and eases off:

  • Delay is multiplied by 1.4 (capped at 2,000 ms)
  • Payloads are reduced to 85% (floor 5)
  • Threads are reduced to 80% (floor 4)

Conversely, if the target responds fast across consecutive modules, the delay is multiplied by 0.9 (floor 100 ms) and threads may grow by 1 (cap 50). Other modes ignore this feedback loop.

JSON output

Pass --json to print the full result tree to stdout, or --json --pretty for human-readable JSON. Use --output <file> (or -o <file>) to also write a pretty-formatted copy to disk — this file is written even on Ctrl+C with the partial result tree.

# To stdout, pretty abspider example.com --modules dns,sslTls --json --pretty # To a file (always pretty) abspider example.com --modules dns,sslTls,ports,cors --output scan.json

The result tree shape:

{ "target": "example.com", "startedAt": "2026-06-07T10:00:00.000Z", "finishedAt": "2026-06-07T10:00:14.000Z", "durationMs": 14000, "mode": "adaptive", "profile": "custom", "modules": { "siteInfo": { "ok": true, "data": { "status": 200, "title": "Example Domain" }, "durationMs": 420 }, "dns": { "ok": true, "data": { "A": ["23.215.0.136"], "AAAA": ["2600:1408:..."] }, "durationMs": 180 }, "whois": { "ok": true, "data": { "registrar": "...", "createdAt": "..." }, "durationMs": 820 }, "sslTls": { "ok": true, "data": { "subject": "CN=example.com", "expiresAt": "..." }, "durationMs": 600 } }, "summary": { "modulesRun": 4, "modulesOk": 4, "modulesFailed": 0, "findings": 0 } }

Field shape varies per module. The JSON is treated as additive — new fields may appear in future releases without a major version bump.

Example: a CORS finding

A real corsMisconfig finding emitted by the CLI in --json mode:

{ "module": "corsMisconfig", "severity": "high", "title": "CORS reflects arbitrary Origin", "description": "The response includes Access-Control-Allow-Origin: <request-origin> with Access-Control-Allow-Credentials: true. This is a classic CORS misconfiguration.", "target": "https://staging.example.com", "evidence": { "requestOrigin": "https://evil.example", "responseHeaders": { "access-control-allow-origin": "https://evil.example", "access-control-allow-credentials": "true" }, "status": 200 }, "remediation": "Reflect only a known allowlist of origins, or set Access-Control-Allow-Origin to a static value and never combine it with Access-Control-Allow-Credentials: true for dynamic origins." }

Graceful shutdown

Press Ctrl+C to stop. The runner:

  1. Cancels the in-flight module with the registered AbortController.
  2. Skips any remaining modules.
  3. Writes a partial result tree to --output if one was provided.
  4. Exits with a non-zero status so CI pipelines can react.

Environment variables

The CLI does not load .env. Only the VirusTotal key is read from the environment.

export VIRUSTOTAL_API_KEY=<your-key> abspider example.com --modules virustotal

Common recipes

# 1. Safe passive-only recon against a public domain abspider example.com # 2. Full coverage on a staging host you own abspider https://staging.example.com --all # 3. Active-only, conservative, JSON to file abspider https://staging.example.com --active --mode conservative --output active.json # 4. Pipe the result into jq for triage abspider example.com --json --pretty | jq '.modules | to_entries | map({module: .key, ok: .value.ok, findings: (.value.findings // 0)})' # 5. Cap a noisy WAF target abspider https://example.com --mode conservative --threads 4 --delay 1500 --payloads 3 # 6. Web port profile only abspider https://example.com --modules ports --port-profile web
