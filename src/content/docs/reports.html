Reports

Every ABSpider Recon scan can be exported as PDF, DOCX, Markdown, CSV, or JSON. Reports are generated client-side in the browser with jsPDF, jspdf-autotable, and docx. The CLI emits JSON via --json --pretty or --output <file>.

Supported formats

FormatBest forGenerated byNotes
PDF Stakeholder deliverables, archival jsPDF + jspdf-autotable Cover page, executive summary, module-by-module findings, evidence, remediation, appendix.
DOCX Editable reports, internal review docx Same content as the PDF, fully editable in Microsoft Word / LibreOffice Writer.
Markdown Wikis, code review, ticketing Hand-rolled serializer in reportService.ts Heading hierarchy, tables, and inline code. Pastes cleanly into GitHub, Notion, and Confluence.
CSV Spreadsheets, statistical analysis Hand-rolled serializer One row per finding with module, severity, confidence, and evidence URL.
JSON Automation, downstream tooling JSON.stringify Full result tree. See the CLI JSON reference.

PDF report layout

The PDF report is the canonical deliverable. Every section can be hidden via the report configuration panel.

Cover page

  • ABSpider Recon branding and logo
  • Target, scan date, scan duration, and operator email
  • Overall security grade (A–F) with a color-coded badge
  • Summary statistics: total findings, modules run, vulnerabilities by severity

Executive summary

  • Plain-language narrative generated from the highest-severity findings
  • Severity breakdown chart (Critical / High / Medium / Low / Info)
  • Top 5 recommendations, ordered by impact

Module-by-module findings

For each module that produced findings, the report includes:

  • Module name and category (passive / active / utility)
  • Severity, confidence, and a short description
  • Evidence (request/response snippets, header values, payload echoes)
  • Proof-of-concept reproduction steps
  • Remediation guidance with code or configuration examples where applicable

Appendix

  • Full scan configuration (modules enabled, mode, threads, payload count)
  • Raw module output (truncated for readability)
  • Methodology and tool versions
  • Authorization reminder and timestamps

Discord embeds

When a scan completes and Discord notifications are enabled, the runner posts a rich embed to the configured webhook. The embed mirrors the executive summary of the PDF.

{ "username": "ABSpider Recon", "embeds": [{ "title": "Scan complete — example.com", "color": 15158332, "fields": [ { "name": "Grade", "value": "B (8.4)", "inline": true }, { "name": "Duration", "value": "14s", "inline": true }, { "name": "Modules", "value": "21 / 21", "inline": true }, { "name": "Findings", "value": "0 critical · 2 high · 1 medium · 3 low" } ], "url": "https://abspider.zanesense.dev/scan/abc123", "timestamp": "2026-06-07T10:00:14.000Z" }] }

The embed color is mapped from the security grade: green for A/B, amber for C/D, red for F.

JSON report

The JSON report contains the full result tree — every field, every payload, every timestamp. It is the source of truth for downstream tools and the input format for any custom report generator.

For the full shape, see CLI JSON output. The dashboard exports the same shape from the Reports page.

CLI outputs

The CLI ships with a clean terminal UI plus structured JSON. The terminal output is not a report — for a deliverable, pipe the JSON into your own template or use the dashboard.

# Pretty JSON to stdout abspider example.com --json --pretty # Pretty JSON to file (preserved on Ctrl+C) abspider example.com --output scan.json # Pipe into a custom Markdown template abspider example.com --json --pretty | node render-md.js > report.md

Customizing the PDF

The PDF layout is configured in Settings → Reports:

  • Include modules — show only findings from the selected modules
  • Severity threshold — exclude findings below a chosen severity (default: low)
  • Detail level — summary, detailed, or forensic
  • Branding — your organization name, contact email, and logo (PNG or SVG)
  • Disclaimer — prepended to the report; useful for client-facing deliverables

Adding your branding

  1. Open Settings → Reports → Branding.
  2. Upload a logo (SVG or PNG, max 2 MB, transparent background recommended).
  3. Enter your organization name, contact email, and website.
  4. Optionally add a custom disclaimer that will be included in the report footer.

Branding is stored per-user in user_settings. There is no global branding — every report reflects the operator who ran the scan.

Retention & deletion

Report source data is stored in Supabase in the user_scans table, scoped to the user that ran the scan. Generated exports are downloaded client-side and are not stored as separate database records. You can:

  • Delete a single scan from the All Scans page
  • Bulk delete scans older than a chosen date
  • Export all of your scans as JSON from Settings → Data Management
  • Delete your account and all associated data from Account → Danger Zone

Automation recipes

Generate a weekly report from a CI job:

# .github/workflows/weekly-report.yml name: Weekly recon report on: schedule: - cron: "0 6 * * 1" # Mondays at 06:00 UTC jobs: scan: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 with: node-version: "20" - run: npm ci - run: npx abspider ${{ secrets.TARGET }} --all --output report.json - uses: actions/upload-artifact@v4 with: name: abspider-report path: report.json
