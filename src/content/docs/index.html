What is ABSpider Recon?

Most public web attacks start with the same recon: a DNS lookup, a certificate transparency query, a port scan, a header review, and a handful of well-known payload checks. ABSpider Recon is a focused, opinionated toolkit that bundles those workflows behind one dashboard and one terminal command. It is built for bug bounty hunters, security engineers, students, and tooling authors.

Authorization is non-negotiable

ABSpider Recon is a security testing tool. It does not grant permission to scan any system. Use it only on assets you own or have explicit written authorization to test. See Security & Legal.

Two Surfaces, One Engine

The platform runs the same scanner engine behind two complementary surfaces, so you can pick the workflow that fits the engagement.

Dashboard

A Vite + React + Supabase app with a polished UI, email/password and magic-link auth, remember-me session control, a first-dashboard beta notice with GitHub issue links, persisted scan history, JSON/PDF/DOCX report export, and per-user settings. The browser is the primary client; Vercel and Docker provide same-origin proxy options for CORS-limited probes.

  • 35 recon modules with a unified card UI
  • Smart scan modes (conservative / adaptive / aggressive)
  • Notification center with Discord webhooks
  • Supabase row-level security for every table

CLI

The published abspider npm package runs the same modules from a terminal — no Supabase session, no third-party keys needed for core modules. Perfect for automation, CI, and air-gapped assessments.

  • One-off: npx abspider example.com
  • Bounded payload counts, delays, and concurrency
  • Graceful Ctrl+C with partial JSON preserved
  • Machine-readable output: --json / --output

Core Capabilities

Privacy-First

Per-user Supabase tables with row-level security. No telemetry, no shared scan results, no phoning home.

Smart Orchestration

The SmartScanManager adjusts payload counts and concurrency based on the target's health and WAF detection.

Bounded by Design

Every active module has explicit payload counts, request delays, and concurrency limits. Nothing fires blindly.

CORS-aware

A tiered request pipeline: direct fetch, then same-origin /api/proxy through the FastAPI backend on Vercel, Docker, or self-hosted deployments.

Multi-Format Reports

Export scan results as styled PDF, DOCX, Markdown, or JSON — generated client-side with jsPDF and docx.

Optional Intel APIs

Pluggable integrations for Shodan, VirusTotal, SecurityTrails, Hunter.io, and Discord webhooks — bring your own key.

System Philosophy

ABSpider Recon is built on three core pillars that guide every architectural decision.

1. Browser-first

Most scanning logic and report generation run in the browser. Supabase handles auth and persistence, while the bounded FastAPI service provides same-origin proxying and protected API-key operations.

2. Smart Orchestration

An adaptive scan engine tunes payload intensity and concurrency in real time. The result: less target noise, fewer false positives, and faster findings.

3. Extensibility

Functionality out of the box. Optional integrations with external intelligence providers enrich the data when you bring your own keys.
