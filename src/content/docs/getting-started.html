Getting Started

Install the dashboard, configure Supabase, and complete your first reconnaissance scan. The CLI works without a Supabase project; the dashboard requires one for authentication and history.

Node.js ≥ 20 npm ≥ 9 Supabase project (free tier OK)

Choose your workflow

ABSpider Recon ships two surfaces backed by the same scanner engine. Pick the path that matches your goal — you can run both side by side.

Dashboard (web app)

The full GUI with email/password or magic-link auth, optional remembered sessions, persistent scan history, and multi-format reports. Requires a Supabase project. Best for interactive assessments and team use.

Dashboard Installation

The dashboard is a Vite SPA that runs the scanner engine in your browser and persists results to Supabase.

  1. Clone the repository

    Grab the source from GitHub and enter the project directory.

    git clone https://github.com/zanesense/abspider-recon.git cd abspider-recon
  2. Install dependencies

    Use the committed lockfile for a reproducible install.

    npm ci # or, if you prefer npm install
  3. Create a Supabase project

    Sign in at supabase.com, create a new project, then copy your Project URL and anon public key from Settings → API.

  4. Configure environment variables

    Copy the template and fill in the two required variables. Anything prefixed VITE_ is exposed to the browser bundle.

    cp .env.example .env # .env is already in .gitignore VITE_SUPABASE_URL=https://your-project-ref.supabase.co VITE_SUPABASE_ANON_KEY=your-anon-key-here # Server-side only; required for dashboard API-key features SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

    Configure provider API keys and Discord webhooks per user in dashboard Settings; never place them in VITE_* variables.

    PowerShell users

    cp is not a native PowerShell cmdlet. Use Copy-Item .env.example .env on Windows PowerShell.

  5. Apply Supabase migrations

    The SQL files in supabase/migrations/ create the schema, row-level security policies, and storage buckets.

    # With the Supabase CLI supabase link --project-ref <YOUR_PROJECT_REF> supabase db push # Or run the files in order in the Supabase SQL Editor: # 1. supabase/migrations/0001_init_schema.sql # 2. supabase/migrations/0002_rls_policies.sql # 3. supabase/migrations/0003_storage_avatars.sql
  6. Start the dev server

    Vite listens on port 5000 by default. Open the URL it prints in your browser.

    npm run dev # → http://localhost:5000
  7. Sign in and start a scan

    Click Login, enter your email, and click the magic link Supabase sends. Once authenticated, navigate to New Scan, enter a target you own, pick a mode, and click Start Reconnaissance.

CLI Installation

The CLI is published as abspider on npm. It bundles the runner and the payload JSON files into a single tarball that works with no Supabase session and no third-party keys for the core modules.

# One-off scan — no install required npx abspider example.com # Or install globally for repeated use npm install -g abspider abspider example.com
bunx abspider example.com # or install globally bun add -g abspider abspider example.com
git clone https://github.com/zanesense/abspider-recon.git cd abspider-recon npm install npm run cli -- example.com

Local development uses the source tree under packages/cli. The published abspider package bundles a frozen copy of the same modules.

CLI environment variables

The CLI only reads the VirusTotal key from your shell environment. It does not load .env.

export VIRUSTOTAL_API_KEY=<your-key> abspider example.com --modules virustotal

All other CLI modules run without configuration.

Your first scan

Pick a target you own or have written permission to test. IANA-owned example.com is a safe choice for the very first run — the Nmap project also offers scanme.nmap.org for port-scanner testing.

Dashboard walkthrough

  1. Sign in

    Use email/password or request a magic link. Keep Remember me checked to persist the session across browser restarts, or uncheck it for a browser-session-only login.

  2. Review the beta notice

    The first dashboard visit shows a beta/dev-stage dialog with links to star the repository or report bugs on GitHub. After dismissal, it is stored locally and will not repeat unless browser storage is cleared.

  3. Open New Scan

    From the sidebar, click New Scan.

  4. Enter the target

    Use a base domain like example.com or a full URL like https://staging.example.com. Subdomains are also accepted.

  5. Pick a mode

    Choose Conservative for low-noise recon, Adaptive (default) for a balance, or Aggressive for the full module set.

  6. Select modules

    Tick the recon modules you want. Active modules are tagged with a red badge — only run them on authorized targets.

  7. Start the scan

    Click Start Reconnaissance. The dashboard polls progress in real time and renders findings as they complete.

  8. Export

    From the Reports page, export the result as PDF, DOCX, Markdown, or JSON.

CLI walkthrough

# Default: passive recon only abspider example.com # Every passive and active CLI module abspider https://example.com --all # Selected modules, save JSON for a downstream tool abspider example.com --modules whois,dns,sslTls,ports,cors --output scan.json # Machine-readable JSON to stdout abspider example.com --modules dns,sslTls --json --pretty

Press Ctrl+C to stop gracefully. The CLI aborts in-flight work, skips remaining modules, and preserves any partial JSON written via --output.
