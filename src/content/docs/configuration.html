Configuration

ABSpider Recon is configured through environment variables, Supabase migrations, and per-user settings stored in the database. This page lists every knob and how it interacts with the dashboard and the CLI.

Environment variables

Only public frontend variables use the VITE_ prefix. Server credentials remain unprefixed and must never be exposed to the browser bundle. Provider keys and Discord webhooks are configured per user in dashboard Settings. The CLI only reads VIRUSTOTAL_API_KEY from your shell environment.

VariableRequiredDefaultDescription
VITE_SUPABASE_URL Yes none Supabase project URL, e.g. https://YOUR-PROJECT-REF.supabase.co.
VITE_SUPABASE_ANON_KEY Yes none Supabase anon publishable key. Protect your data with Row Level Security.
SUPABASE_SERVICE_ROLE_KEY API-key features none Server-side credential used by FastAPI for authenticated API-key storage and provider requests.
SUPABASE_URL optional VITE_SUPABASE_URL Optional server-side Supabase URL override for FastAPI.
VITE_DEV_MODE Never false Enables verbose dev tooling and HMR overlays.
VITE_LOG_LEVEL Never info One of debug, info, warn, error.

Start from the template

cp .env.example .env # .env is already in .gitignore
PowerShell users

cp is not a native PowerShell cmdlet. Use Copy-Item .env.example .env on Windows PowerShell.

Supabase setup

Apply the migrations in supabase/migrations/ before using the dashboard end to end. Each file is idempotent and can be re-run.

# With the Supabase CLI supabase link --project-ref <YOUR_PROJECT_REF> supabase db push # Or run the files in order in the Supabase SQL Editor: # 1. supabase/migrations/0001_init_schema.sql # 2. supabase/migrations/0002_rls_policies.sql # 3. supabase/migrations/0003_storage_avatars.sql

Dashboard settings

The dashboard exposes only settings with active runtime consumers. The Discord webhook and proxy list are stored in user_settings; provider keys are managed by the authenticated FastAPI backend. Scan threads, modules, payload counts, and proxy usage are selected for each scan on the New Scan page.

Theme

The light/dark theme control is available in the application navigation and is remembered in localStorage under abspider-theme.

Account settings

Account Settings manages Supabase user metadata, avatar upload, password changes, data export, and permanent account deletion. Account deletion is authenticated by the FastAPI backend and uses the server-side Supabase service role; the key is never exposed to the browser.

Proxy configuration

Configure multiple proxies for automatic rotation. The dashboard uses them in a round-robin or random strategy to avoid rate limiting.

{ "proxies": [ "http://proxy1.example.com:8080", "http://user:pass@proxy2.example.com:3128", "socks5://proxy3.example.com:1080" ], "rotationStrategy": "round-robin", "proxyTimeout": 5000, "validateProxies": true, "fallbackToDirect": true }

Supported schemes: http, https, socks5. Authentication is supported via the user:pass@ URL syntax.

Discord webhooks

Send scan-completion alerts to a Discord channel. The webhook URL is stored in user_settings and treated as a secret.

  1. Open Discord → Server Settings → Integrations → Create Webhook.
  2. Name it (e.g. "ABSpider Notifications") and pick the channel.
  3. Copy the URL — it must start with https://discord.com/api/webhooks/.
  4. In the dashboard, go to Settings → Notifications, paste the URL, and click Test Webhook.
{ "webhookUrl": "https://discord.com/api/webhooks/...", "enableNotifications": true, "notifyOnComplete": true, "notifyOnError": true, "notifyOnVulnerability": true, "minimumSeverity": "medium" }

Webhook payloads include a rich embed: target, duration, modules executed, vulnerabilities found (with severity), and a deep link to the dashboard report.

Server-side API key access

Provider keys are stored in user_api_keys. Migration 0004_secure_api_keys.sql removes direct browser access; authenticated FastAPI endpoints use the Supabase service role to read and update the current user's key bundle. Never expose SUPABASE_SERVICE_ROLE_KEY through a VITE_* variable.

Scan profiles

Save frequently used configurations as templates. The current implementation stores them under scan-templates in browser localStorage and loads the most recently saved template from the New Scan page. Templates do not synchronize across browsers.

CORS bypass configuration

The dashboard uses a tiered request pipeline. Vercel Services and Docker/local backend deployments use the FastAPI proxy at the same /api/proxy route.

{ "primaryProxy": "https://abspider-recon.vercel.app/api/proxy?url=", "fallbackProxies": [ "https://api.allorigins.win/raw?url=", "https://corsproxy.io/?" ], "enableCorsWorkaround": true }

Use the proxy only for hosts you are authorized to test. It is not a generic web proxy.
