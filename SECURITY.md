# Security Policy

## Supported Versions

The following table lists which versions of ABSpider Recon currently receive
security updates. Older releases are not maintained and should be upgraded.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest| :x:                |

## Reporting a Vulnerability

Please report security issues **privately** so we can investigate and ship a
fix before public disclosure. Do not open a public GitHub issue for suspected
vulnerabilities.

### Contact

- GitHub Security Advisories: https://github.com/zanesense/abspider-recon/security/advisories/new
- Email: security@zanesense.dev

### What to include

A useful report contains:

1. A clear description of the issue and its impact.
2. Steps to reproduce, including a proof-of-concept or payload when safe.
3. The affected version, commit SHA, or release tag.
4. Your name / handle (optional) if you would like to be credited in the
   advisory.

## Response Timeline

We aim to follow this rough timeline once a report is acknowledged:

| Stage                       | Target time |
| --------------------------- | ----------- |
| Initial acknowledgement     | 3 business days |
| Triage and severity rating  | 7 business days |
| Patch development           | 30 days     |
| Public disclosure           | After a fix is available, or 90 days, whichever comes first |

We will keep you informed of progress and, with your permission, credit you
in the release notes and security advisory.

## Safe Harbour

We will not pursue legal action against researchers who:

- Make a good-faith effort to avoid privacy violations, data destruction,
  or service disruption.
- Only interact with accounts they own or are explicitly authorised to test.
- Stop testing immediately if they encounter user data and report it to us
  so we can take next steps together.

Thank you for helping keep ABSpider Recon and its users safe.
